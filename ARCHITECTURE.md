## 2. Технические решения

### Backend: Next.js API Routes (а не NestJS)
**Почему:** Единый деплой, SSR+API в одном проекте, проще инфраструктура.
Для B2B платформы с ~50k товаров API Routes достаточно.

### Поиск: PostgreSQL full-text search + pg_trgm
**Почему:** Не нужен отдельный сервис. PostgreSQL GIN-индексы + триграммы
дают быстрый fuzzy-поиск по артикулам и названиям. Для 50k товаров хватит.
Meilisearch добавим позже если понадобится.

### Хранилище: MinIO (self-hosted S3)
**Почему:** Бесплатно, совместимо с AWS S3 API, можно потом мигрировать.

## 4. Ключевые потоки

### Регистрация
User → Форма → status=PENDING → Admin видит в панели → Approve → status=ACTIVE

### Формирование КП
Клиент → Добавляет товары → "Сформировать КП" → PDF с логотипом, ценами, реквизитами

### Импорт товаров
Admin → Загружает XML → Маппинг колонок → Превью → Импорт → Матчинг по SKU/модели → Обновление БД

### Импорт цен
Admin → Загружает XLSX → Маппинг → Матчинг по артикулу → Обновление цен и остатков
