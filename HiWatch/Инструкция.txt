Ты senior Python developer и data engineer. Нужно создать готовый парсер каталога с сайта поставщика, который обойдет все категории и все карточки товаров, соберет полный список товаров без цен и сохранит данные в структурированной файловой системе.

## Цель

Нужно получить локальную базу товаров со следующими данными по каждой позиции:

* название товара
* модель
* категория / подкатегория
* основное фото товара + дополнительные фото если есть
* описание товара
* технические характеристики
* паспорт / datasheet / инструкция / сертификат / pdf-документ, если он есть на странице
* ссылка на страницу товара

## Важно

* Цены собирать не нужно
* Нужно пройтись по всем вкладкам / категориям / подкатегориям
* Нужно собрать все товары сайта, а не только первую страницу
* Описание и технические характеристики нужно сохранить в XML
* Для каждого товара должна создаваться отдельная папка
* Парсер должен быть устойчивым, аккуратным и пригодным для реального использования

---

## Требуемая структура папок

Нужно сохранить всё в таком виде:

```text
НовыйБренд/
├── output/
│   ├── all_products.xml
│   ├── all_products.csv
│   ├── parser.log
│   ├── Категория 1/
│   │   ├── Модель-1/
│   │   │   ├── product.xml
│   │   │   ├── image.jpg
│   │   │   ├── image_2.jpg       (если есть доп. фото)
│   │   │   ├── паспорт.pdf       (если есть)
│   │   │   ├── сертификат.pdf    (если есть)
│   │   │   └── инструкция.pdf    (если есть)
│   │   ├── Модель-2/
│   │   │   └── ...
│   ├── Категория 2/
│   │   └── ...
```

Если у товара несколько фото:
* главное фото — image.jpg
* дополнительные — image_2.jpg, image_3.jpg

Если PDF несколько:
* называть по типу: паспорт.pdf, сертификат.pdf, инструкция.pdf
* если тип неизвестен: document.pdf, document_2.pdf

Если паспорта нет:
* просто не создавай pdf-файл

## Что должно быть внутри product.xml

Для каждого товара нужно создать файл product.xml:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<product>
    <name>Полное название товара</name>
    <model>DS-I200</model>
    <category>IP камеры</category>
    <subcategory>Купольные камеры</subcategory>
    <product_url>https://сайт.com/товар</product_url>

    <description>
        <paragraph>Текст описания 1</paragraph>
        <paragraph>Текст описания 2</paragraph>
    </description>

    <specifications>
        <spec>
            <name>Разрешение</name>
            <value>2 Мп</value>
        </spec>
        <spec>
            <name>Объектив</name>
            <value>2.8 мм</value>
        </spec>
    </specifications>

    <files>
        <image>image.jpg</image>
        <image>image_2.jpg</image>
        <passport>паспорт.pdf</passport>
        <passport>сертификат.pdf</passport>
    </files>
</product>
```

## Логика парсинга

1. Зайти на сайт
2. Найти все товарные категории (из меню, sidebar или карты сайта)
3. В каждой категории найти все товары (с учётом пагинации)
4. Открыть каждую карточку товара
5. Вытащить: название, модель, категорию, описание, характеристики, фото, PDF
6. Скачать файлы
7. Сформировать XML
8. Разложить всё по папкам

## Технические требования к реализации

Проект на Python. Модульная структура:

```text
НовыйБренд/
├── main.py              # Главный файл запуска
├── parser.py            # Логика парсинга (категории, товары, спеки)
├── downloader.py        # Многопоточное скачивание файлов
├── xml_builder.py       # Генерация XML и CSV
├── utils.py             # HTTP-запросы, прогресс-бар, утилиты
├── requirements.txt     # Зависимости
└── README.md
```

Библиотеки:
* requests + BeautifulSoup4 + lxml — основной парсинг
* concurrent.futures (ThreadPoolExecutor) — параллельное скачивание
* logging — логирование в файл и консоль

Если сайт рендерит контент через JavaScript:
* сначала проверь обычный HTML (requests)
* если данных в HTML нет — используй Playwright

## Требования к коду

* Многопоточный парсинг товаров (10-15 потоков)
* Многопоточное скачивание файлов (10-20 потоков)
* Прогресс-бар в консоли с ETA
* Повторные попытки при ошибках сети (3 retries)
* Таймауты на все запросы
* Нормализация имён папок (убирать запрещённые символы)
* Пропуск дублей
* Кодировка UTF-8
* Возможность продолжить, если часть данных уже скачана
* --no-download флаг для запуска без скачивания файлов
* Не собирать цены
* Не ломаться если у товара нет PDF / фото / части характеристик
* Логировать какие товары обработаны, а какие нет

## Совместимость с системой импорта

Результат парсера должен быть совместим с системой импорта сайта VidosGroup.
Импортёр читает product.xml из папки output/ и понимает оба формата:

Формат тегов:
* <name> — название товара (обязательно)
* <model> или <article> — модель/артикул (обязательно, используется как уникальный ключ)
* <brand> — бренд
* <category> — категория
* <subcategory> — подкатегория
* <description> — описание (внутри <paragraph>)
* <specifications> — характеристики (<spec><name>...<value>...</spec>)
* <files> — файлы:
  * <image> — имя файла фото (image.jpg) или URL
  * <passport> — имя PDF файла (паспорт.pdf, сертификат.pdf)
* <product_url> — ссылка на оригинальную страницу

PDF файлы должны лежать рядом с product.xml в папке товара.
Импортёр ищет все *.pdf в папке и определяет тип по имени файла:
* паспорт/passport → тип "passport"
* сертификат/certificate → тип "certificate"
* инструкция/instruction/manual → тип "instruction"

После парсинга папка автоматически появится в админ-панели сайта
(http://localhost:3000/admin/import → вкладка "Папка бренда")
и можно будет импортировать товары одной кнопкой.
